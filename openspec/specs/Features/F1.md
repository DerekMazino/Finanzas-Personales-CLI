### Feature: Gestion de periodos
## HU1: Agregar periodos nuevos
# Descripción
- **Como** Usuario 
- **Requiero** Agregar un nuevo periodo
- **Para así** Poder agregar ingresos y egresos en dicho periodo
# Criterios
- **Criterio 1** Dado que el usuario seleccionó la opción para ingresar un nuevo periodo, si no existe un periodo del mes actual, el sistema le sugerirá al usuario que si desea crearlo, dando la opción S - Si o N - No.
- **Criterio 1.1** Si el cliente dice que si, el programa debe:
    - Crear el periodo.
    - **Copiar todos los conceptos marcados como Recurrentes** de periodos anteriores a este nuevo periodo (con valor inicial basado en el último registro o 0.0 según se defina en la implementación).
- **Criterio 2** Dado que el usuario seleccionó la opción para ingresar un nuevo periodo, dado que cada periodo es unico, si se intenta crear un periodo que ya existe, el programa debe informar que esto no es posible ya que el registro ya existe.

## HU2: Listar periodos
# Descripción
- **Como** Usuario 
- **Requiero** Listar todos los periodos disponibles
- **Para así** Poder seleccionar el periodo donde quiero gestionar mis ingresos y egresos.
# Criterios
- **Criterio 1** Dado que el usuario seleccionó la opción para ver los periodos registrados, el sistema debe mostrarle un listado de estos.
- **Criterio 2** Dado que el usuario seleccionó la opción para ver los periodos registrados, si no hay ninguno, el sistema debe mostrarle un mensaje indicando que no existe ningún periodo y que puede crear uno.
